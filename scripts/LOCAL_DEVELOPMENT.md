# 💻 Инструкция по локальной настройке (Windows)

Для быстрой разработки втроем лучше всего использовать **Гибридный подход**:
- **База данных и Redis**: запускаются в Docker (чтобы не устанавливать их на Windows).
- **Сайт и Бот**: запускаются через Node.js (для мгновенного применения изменений).

---

## 🏗️ 1. Подготовка системы

Убедитесь, что на вашем ПК установлены:
1. **Node.js 22+**: [Скачать](https://nodejs.org/)
2. **Docker Desktop**: [Скачать](https://www.docker.com/products/docker-desktop/) (убедитесь, что он запущен).

---

## 🛠️ 2. Запуск инфраструктуры

В терминале проекта (D:\Smmplan) запустите только базу и редис:
```powershell
docker compose up -d db redis
```
Это откроет порты:
- **PostgreSQL**: `localhost:5433`
- **Redis**: `localhost:6380`

---

## ⚙️ 3. Настройка окружения (.env)

Создайте или обновите `.env` файл для локальной работы:
```env
# Подключение к локальному Docker
DATABASE_URL="postgresql://smmuser:smmpassword@localhost:5433/smmplan?schema=public"
REDIS_URL="redis://localhost:6380"

# Остальные настройки (токены бота и т.д.)
BOT_TOKEN="ВАШ_ТОКЕН"
NODE_ENV="development"
```

---

## 📦 4. Установка зависимостей и БД

Выполните команды по очереди:
```powershell
# Установка библиотек
npm install

# Создание таблиц в локальной базе
npx prisma migrate dev

# (Опционально) Наполнение базы тестовыми данными
npx tsx scripts/seed-real-data.ts
```

---

## 🚀 5. Запуск для разработки

Откройте два окна терминала:

**Окно 1 (Сайт):**
```powershell
npm run dev
```
Сайт будет доступен по адресу: `http://localhost:3000`

**Окно 2 (Бот):**
```powershell
npm run bot:dev
```
Бот будет реагировать на сообщения в Telegram в реальном времени.

---

## ✅ 6. Как проверить перед отправкой на сервер?

Перед тем как делать `git push`, всегда проверяйте две вещи:
1. **Линтер**: `npm run lint` (не должно быть красных ошибок).
2. **Сборка**: `npm run build` (проект должен успешно собраться локально).

---

## 🚢 7. Загрузка на сервер

Когда вы всё проверили и запушили код в GitLab:
1. Если настроен CI/CD (см. `COLLABORATION_GUIDE.md`) — сервер обновится **автоматически**.
2. Если обновляете вручную: см. `DEPLOY_2.md` (Шаг 6).
