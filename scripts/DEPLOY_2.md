# 🚀 Инструкция по развёртыванию Smmplan (для новичков)

Эта инструкция поможет вам запустить проект на чистом сервере (VPS) с 4GB RAM, используя Windows и WinSCP.

---

## 📋 Шаг 0: Подготовка сервера

Если на вашем сервере ещё нет Docker, выполните эти команды в консоли сервера (через Putty или терминал WinSCP):

```bash
# Обновляем систему
sudo apt update && sudo apt upgrade -y

# Устанавливаем Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# Проверяем, что Docker работает
docker --version
docker compose version
```

---

## 💻 Шаг 1: Подготовка файлов на вашем ПК

Убедитесь, что в папке `D:\Smmplan` у вас есть следующие файлы:
1. `smmplan-app.tar` (Образ сайта)
2. `smmplan-bot.tar` (Образ бота)
3. `docker-compose.prod.yml` (Главный файл настроек)
4. `.env` (Ваши ключи и пароли)
5. Папка `nginx` (внутри должен быть `nginx.conf`)
6. Папка `postgres` (внутри должен быть `tune.sql`)

---

## 📂 Шаг 2: Копирование через WinSCP

1. Подключитесь к серверу через WinSCP.
2. В правой панели (сервер) перейдите в папку `/opt`.
3. Нажмите правую кнопку мыши -> **Создать** -> **Директория**. Назовите её `smmplan`.
4. Зайдите внутрь `/opt/smmplan`.
5. Перетащите все файлы из **Шага 1** в эту папку.
   - *Важно:* Папки `nginx` и `postgres` должны скопироваться целиком вместе с файлами внутри.

---

## 🛠️ Шаг 3: Запуск на сервере

Теперь откройте терминал сервера (кнопка "Терминал" в WinSCP или через Putty) и вводите команды по очереди:

### 1. Переходим в папку с проектом
```bash
cd /opt/smmplan
```

### 2. Загружаем образы в Docker (это может занять 2-5 минут)
```bash
docker load -i smmplan-app.tar
docker load -i smmplan-bot.tar
```

### 3. Запускаем все службы
```bash
docker compose -f docker-compose.prod.yml up -d
```
*После этой команды подождите ~30 секунд, пока база данных "прогреется".*

---

## 🗄️ Шаг 4: Настройка базы данных и Админки

Это нужно сделать только **один раз** при первой установке.

### 1. Создаем таблицы в базе
```bash
docker exec -it smmplan-bot npx prisma migrate deploy --schema=prisma/schema.prisma
```

### 2. Создаем администратора
```bash
docker exec -it smmplan-bot npx tsx scripts/create-first-admin.ts
```

2. Перейдите по адресу: `http://ваш-ip/login`

## 🆘 Если что-то не работает

| Проблема | Что делать? |
| :--- | :--- |
| Сайт не открывается | Проверьте порты: `ufw allow 80/tcp` и `ufw allow 443/tcp` |
| Ошибка в логах бота | Проверьте `BOT_TOKEN` в файле `.env` |
| Мало памяти | Включите Swap (инструкция ниже) |

### Как посмотреть логи (если есть ошибки):
```bash
docker logs smmplan-app --tail 50  # Логи сайта
docker logs smmplan-bot --tail 50  # Логи бота
```

## 🔄 Шаг 6: Как обновлять проект (самое важное)

Если вы изменили код или настройки на своем ПК и хотите отправить их на сервер:

### 1. Какие файлы менять?
- **Если изменили код (src/, scripts/):** Вам нужно заново собрать образы (`.tar` файлы) и перекинуть их.
- **Если изменили `.env`:** Просто скопируйте обновленный файл `.env` через WinSCP и перезапустите контейнеры.
- **Если изменили `nginx.conf`:** Скопируйте его в папку `nginx/` на сервере и перезапустите Nginx.

### 2. Как "запулить" исправления (Manual Flow):
1. **На вашем ПК** соберите новые образы (команды из консоли в папке проекта):
   ```powershell
   # Собираем и сохраняем сайт
   docker build -t smmplan-app .
   docker save smmplan-app > smmplan-app.tar

   # Собираем и сохраняем бота
   docker build -t smmplan-bot --target bot-runner .
   docker save smmplan-bot > smmplan-bot.tar
   ```
2. **Через WinSCP** удалите старые `.tar` на сервере и залейте новые.
3. **На сервере** выполните команду из папки `/opt/smmplan`:
   ```bash
   docker load -i smmplan-app.tar
   docker load -i smmplan-bot.tar
   docker compose -f docker-compose.prod.yml up -d --remove-orphans
   ```

## 🏗️ Профессиональный подход: GitLab CI/CD

Если вы используете **GitLab**, то "правильный" способ — это автоматизация.

**Как это работает:**
1. Вы делаете `git push` своего кода в GitLab.
2. GitLab Runner (в облаке или на мощном ПК) автоматически собирает Docker-образ.
3. Образ сохраняется в **GitLab Container Registry**.
4. На сервере вы просто пишете:
   ```bash
   docker compose -f docker-compose.prod.yml pull
   docker compose -f docker-compose.prod.yml up -d
   ```
   *Это гораздо быстрее, так как не нужно передавать тяжелые `.tar` файлы вручную.*

**Почему не GitLab сейчас?**
Для вашего сервера (4GB) запуск "сборки" внутри GitLab Runner прямо на этом же сервере может его "уронить" (закончится память). Если вы хотите настроить это — дайте знать, я помогу составить `.gitlab-ci.yml`.
