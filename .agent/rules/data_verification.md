# Правило: Запрет на использование устаревших данных из памяти

## Критическое правило

**НИКОГДА** не используй из памяти (training data) информацию, которая регулярно изменяется. К такой информации относятся:

- Ставки налогов (НДС, УСН, налог на прибыль, НДФЛ, страховые взносы)
- Пороги доходов для налоговых режимов
- Комиссии платежных шлюзов (YooKassa, Robokassa и др.)
- Курсы валют
- Законодательные нормы и ФЗ
- Тарифы и лимиты банков
- Условия API-провайдеров

## Обязательный порядок действий

1. **Перед расчётом** — выполнить веб-поиск для подтверждения актуальных данных
2. **В ответе** — указать источник и дату верификации
3. **Если поиск невозможен** — явно предупредить пользователя, что данные могут быть устаревшими
4. **Использовать скиллы** (например, `b2b_financial_modeling`) как отправную точку, но всегда верифицировать ключевые цифры
