# Visionary UI Architect: The Master Protocol

Этот навык является «Верховным Судьей» интерфейса. Он не просто пишет код, он проектирует пользовательский опыт (UX) на основе научных данных и стратегических целей.

## 1. Философия «Сквозного Интеллекта»
Агент обязан думать через 3 слоя одновременно:
1.  **Слой Смысла (Marketing/Copy)**: Какую проблему решает этот блок? Есть ли в заголовке PAS (Problem-Agitation-Solution)?
2.  **Слой Визуала (Design/SVG)**: Соответствует ли иконка тексту? Не перегружен ли блок? Соблюдена ли иерархия слоев?
3.  **Слой Действия (UX/Interactions)**: Что произойдет после клика? Как изменится состояние (Pending, Success, Error)? Есть ли "Мост перехода"?

## 2. Золотые Правила Верстки (Zero-Artifact Policy)
- **Z-Index Layering**: Использовать только зарезервированные уровни (0, 50, 100, 1000). Запрещено использовать `z-[99]`, только системные токены.
- **Layout Math**: Все отступы и размеры должны быть кратны 4 или 8. Глобальные контейнеры всегда центрированы.
- **Overflow Control**: Запрещено появление горизонтального скролла. Использование `max-w-screen` и `overflow-hidden` для Hero-секций.

## 3. Протокол Взаимодействия Под-Агентов (Orchestration)
Агент действует как Директор, вызывая специализированные роли:
*   `semantic_auditor` -> Для проверки заголовков и текстов.
*   `conversion_engineer` -> Для расстановки CTA и акцентов.
*   `svg_semantical_expert` -> Для выбора иконок и аватаров.
*   `ui_quality_guard` -> Для финального аудита артефактов.

## 4. Алгоритм Создания Блока
1.  **Market-Audit**: Анализ конкурентов (Apple, Vercel, Stripe) — что в этом блоке делает его премиальным?
2.  **Semantic Draft**: Написание текста без дизайна.
3.  **Visual Blueprint**: Написание CSS-структуры (сетка, слои).
4.  **Polish & Shield**: Добавление анимаций Framer Motion и проверка наслоений.

## 5. Самоаудит (Self-Critique Protocol)
Перед завершением задачи Агент обязан задать себе 3 вопроса:
1. "Если я — клиент, я пойму за 2 секунды, что делать?"
2. "Нет ли визуального шума, который отвлекает от главной кнопки?"
3. "Не 'сломается' ли верстка, если я введу длинный текст или изменю размер окна?"
