# Smmplan Development Plan: Link Analysis & Anti-Fraud Architecture

Этот документ содержит план внедрения наработок из набора навыков (Telegram, VK, Instagram, YouTube, Twitch, TikTok) в архитектуру проекта Smmplan.

## Задача 1: Unified Link Analyzer Service
**Файл**: `src/services/core/link-analyzer.service.ts`
- Создать единый сервис для анализа и очистки ссылок, который будет вызываться при создании каждого заказа.
- **Функционал**:
  - `cleanTrackingParameters(url)`: Удаление мусора (`?igsh=`, `?_t=`, `utm_*`).
  - `detectPlatform(url)`: Определение соцсети (включая Discord, Threads и МАКС).
  - `extractId(url)`: Извлечение унифицированного идентификатора (19 цифр для TikTok, 11 для YT, Invite Code для Discord).
  - `resolveShortLink(url)`: Развертывание `vm.tiktok.com`, `youtu.be`, `ig.me`, `discord.gg` в прямые ссылки через HTTP HEAD.

## Задача 2: Асинхронные валидаторы (Deep Checkers)
- Интегрировать микро-проверки "здоровья" сущностей *ДО* отправки заказа SMM-провайдеру.
- **VK**: Проверка открытого списка участников сообщества, замер "собачек".
- **Twitch**: Проверка статуса "Live" через CDN-картинки превью.
- **Instagram/Threads**: Поиск мета-тега `og:description` для проверки приватности аккаунта.
- **Discord**: Валидация инвайт-кода через Discord API (`/invites/{code}`) на предмет срока действия.

## Задача 3: Provider Link Translation Matrix
**Файл**: `src/services/providers/link-translator.service.ts`
- **Проблема**: Провайдеры (VexBoost, API-X и т.д.) требуют разные форматы ссылок для одних и тех же услуг.
- **Решение**: Создание адаптера трансформации. 
  - На вход адаптер получает очищенную "Идеальную Ссылку" и ID сервиса провайдера. 
  - На выход адаптер выдает строку, ожидаемую конкретно этим провайдером (например, для одного провайдера это прямая ссылка, для другого только `Media ID` из Base62).

## Задача 4: Рекомендательная UX-система предупреждений (Frontend Alerting)
- Изменение логики создания заказа клиентом в панели управления:
  - Если `link-analyzer.service.ts` отвечает, что в группе ВК меньше 10 человек или список участников скрыт, клиент моментально видит `Warning: Накрутка невозможна, список участников скрыт. Заказ будет отклонен провайдером. Откройте участников и повторите попытку.`
  - Показ предупреждения о Теневом бане (Shadowban/Red Mark) для Telegram, если клиент заказывает вступления на инвайт-ссылку.
  - Рекомендация добавить услугу Чат-ботов (Chat Mimicry) при заказе просмотров на Twitch (если заказано >500 вьюверов).
