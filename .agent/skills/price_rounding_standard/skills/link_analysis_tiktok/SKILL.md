# TikTok Link Analysis Skill (v1.0 - The ByteDance Matrix)

Глубокий разбор структуры ссылок TikTok, алгоритмов удержания, "теневых банов" (Shadowban) и методов обхода Device Fingerprinting на 2025/2026 год.

## 1. Анатомия и паттерны ссылок (Stage 1)

Алгоритм TikTok (ByteDance) генерирует разные форматы ссылок в зависимости от устройства (iOS, Android, Web).

### Основные форматы ссылок (Regex Pattern):

1. **Short Mobile Links (Share Links)**:
   - Форматы: `vm.tiktok.com/xxxx/` или `vt.tiktok.com/xxxx/`
   - Это редирект-ссылки. Сервер SMM-панели должен выполнить HEAD-запрос для расшифровки конечного `video_id`.
   - **Regex**: `v[mt]\.tiktok\.com\/([a-zA-Z0-9]+)`

2. **Web Browser (Full URLs)**:
   - Формат: `tiktok.com/@username/video/1234567890123456789`
   - Самый надежный формат для SMM. Длина ID обычно 19 цифр.
   - **Regex**: `tiktok\.com\/@[a-zA-Z0-9_.-]+\/video\/(\d+)`

3. **Tracking Parameters (The Spyglass)**:
   - TikTok добавляет параметры: `?_t=8Zyqw...&_r=1`
   - `_t=` — это временная метка (Timestamp) и криптографическая соль сессии шаринга.
   - ОЧИСТКА ОБЯЗАТЕЛЬНА: Переход 1000 ботов по ссылке с одним и тем же `_t=` мгновенно "палит" сетку ботов, так как этот параметр должен быть уникальным для каждой новой генерации ссылки "Поделиться".

## 2. Системы детекции ботов в 2025 (Anti-Fraud & Shadowban)

TikTok обладает одним из самых свирепых ИИ-антифродов (Advanced Engagement Scoring System).

| Метрика | Как детектирует бота | Рекомендация для SMM |
| :--- | :--- | :--- |
| **Complete/Watch Time Ratio** | 1000 просмотров с `Watch Time = 0.5s` (мгновенный клик-офф). Итог: Алгоритм помечает видео как "отброс" и режет охват. | **Retention View bots**: Просмотр должен длиться дольше 15-20 секунд, а для коротких видео — эмулироваться зацикливание (Looping). |
| **Device Fingerprinting** | Переход с серверных IP (AWS, DigitalOcean) или без мобильных отпечатков. | Использование **только мобильных прокси** с подменой Canvas, WebRTC и Gyroscope data (данные гироскопа, которые TikTok собирает даже в браузере). |
| **Referral Source Tracking** | Трафик приходит из `Unknown` источников (прямые GET-запросы скрипта). | **Referer Spoofing**: Подмена источника на `whatsapp.com`, `instagram.com` или внутреннюю ленту `For You`. |

## 3. Реверс-инжиниринг: Как избежать "Теневого бана" (Shadowban Avoidance)

Если провайдер некачественный, видео клиента не просто не взлетит, оно получит "Скрытый бан" (0 просмотров в ленте "Рекомендации").

### A. Спам-Циклы (Engagement Phishing)
- **Токсичные действия**: "Лайк без просмотра" или "Комментарий на 1 секунде видео". 
- **Решение (Behavioral Delay)**: Лайк должен ставиться случайным образом в диапазоне от 50% до 80% времени проигрывания видео. Комментарий "пишется" (Keyboard Event Emulation), а не вставляется моментально через буфер.

### B. Traffic Laundering (Отмывание просмотров)
В алгоритмах TikTok 2025/2026 искусственно завышенные счетчики игнорируются при ранжировании в "Рекомендациях" (For You Page). 
- **SEO & Search Discovery**: Лучший метод накрутки — это когда юзербот вводит в поисковике TikTok ключевые слова из описания видео клиента, "находит" его в ленте поиска и смотрит.
- **Profile Routing**: Бот заходит на страницу автора, скроллит вниз и открывает нужное видео оттуда.

## 4. Алгоритм для SMM-панели (Best Practice)
1. **URL Unwrap**: Если клиент ввел `vm.tiktok.com`, сервер обязательно делает curl/HEAD запрос, ловит 302/301 Redirect и достает финальный 19-значный `video_id`.
2. **Purge Tracker Query**: Безжалостно отрезать от ссылки `?_t=`, `?_r=` и прочий мусор. Оставить только чистую структуру.
3. **No-Bot Checking**: Проверка видео на блокировку (страница удалена / видео приватно) ДО старта заказа.
4. **Volume Alerting**: Запрет на старт 10,000 просмотров за 1 час на аккаунт, у которого нет ни одного organic видео с 500+ просмотрами. (Оповещение клиента о риске теневого бана).
