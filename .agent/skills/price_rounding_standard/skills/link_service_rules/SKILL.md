# Матрица совместимости ссылок и услуг (v1.0)

Это руководство определяет, какие типы ссылок (Link Object Types) подходят для каких категорий услуг. Нарушение этой матрицы ведет к ошибкам выполнения на стороне провайдеров.

## 1. Telegram

| **TG_POST** (t.me/user/1) | `COMMENTS`, `REACTIONS`, `REPOSTS`, `STARS` (на пост), `VIEWS` (на 1 пост) | Целевое действие на контент. |
| **TG_CHANNEL** / **TG_CHAT** (t.me/user) | `SUBSCRIBERS`, `BOOSTS`, `VIEWS` (авто), `REACTIONS` (авто) | Услуги на канал/группу. |
| **TG_STORY** (t.me/user/s/1) | `STORIES` (просмотры историй) | Только прямые ссылки на сторис. |
| **TG_PROFILE** (t.me/user) | `STARS` (подарки/на профиль) | Звезды конкретному юзеру. |
| **TG_BOT** (t.me/botname) | `REFERRALS`, `OTHER` | Услуги для ботов (старты, рефералы). |
| **TG_INVITE** (t.me/+) | `SUBSCRIBERS` | Приватные инвайт-ссылки. Требуют выбора (Канал или Группа) и предупреждения. |

## 2. Общие принципы взаимодействия (Core Logic)

1. **Техническое соответствие (targetType)**:
   - `POST` ссылки -> Целевые действия (`targetType = POST`).
   - `CHANNEL/PROFILE` ссылки -> Аудиторные действия (`targetType = CHANNEL`).
2. **Случаи "Cross-Matching"**:
   - Авто-услуги (Авто-реакции, Авто-просмотры) всегда требуют ссылку на **CHANNEL**, даже если их результат появляется на будущих **POST**.
3. **Ошибки пользователя**:
   - Если пользователь вставляет ссылку на пост для покупки "Подписчиков", система должна выдать предупреждение или заблокировать выбор.
