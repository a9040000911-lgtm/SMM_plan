# Software Gold Standard (SGS)

Данный навык (Skill) определяет архитектурные, инженерные и этические правила разработки в проекте Smmplan. Любое изменение в коде должно проходить внутреннюю проверку на соответствие этим стандартам.

## 1. Архитектурные принципы (Clean & Scalable)
- **Separation of Concerns (SoC)**: Четкое разделение на UI (Stitch), Бизнес-логику (Services) и Доступ к данным (Prisma).
- **Stateless Logic**: Серверные функции и API должны быть максимально независимыми и предсказуемыми.
- **Idempotency**: Повторные вызовы API/Actions (например, создание заказа) не должны приводить к дублированию данных или финансовым ошибкам.

## 2. Безопасность (Security First - OWASP Based)
- **Zero Trust**: Каждая Server Action и API эндпоинт должны проверять авторизацию и права доступа (RBAC) заново, даже если пользователь залогинен.
- **Data Sanitization**: Весь ввод от пользователя (ссылки, описания) должен санироваться перед записью в БД или рендерингом в боте.
- **Sensitive Data**: Ключи API, пароли и секреты никогда не должны храниться в открытом виде. Использовать хеширование (argon2/bcrypt) или маскирование при выводе в UI.
- **Id Isolation**: Все запросы к БД должны жестко фильтроваться по `projectId` и `userId`.

## 3. Искусственный Интеллект и Автоматизация (AI-Driven Quality)
- **Self-Healing Code**: При обнаружении ошибок (линтер, тесты, падение сборки) агент обязан самостоятельно проанализировать логи и исправить код, не дожидаясь тикета от пользователя.
- **Predictive Validation**: Использовать логику "умных подсказок" при вводе данных пользователем (напр. распознавание платформы по ссылке).
- **LLM-Context Harmony**: Код должен быть написан так, чтобы его было легко понимать как человеку, так и ИИ (четкая типизация, JSDoc, отсутствие чрезмерной вложенности).

## 4. UI/UX Стандарты (Premium Experience)
- **Stitch Methodology**: Сборка интерфейсов из атомарных компонентов с использованием Tailwind 4 и Framer Motion.
- **Micro-interactions**: Каждый клик/действие пользователя должно подтверждаться визуально (скелетоны, лоадеры, тосты).
- **Mobile First**: Дизайн должен быть удобен для управления пальцем. Кнопки не менее 44x44px, важные элементы в зоне досягаемости большого пальца.
- **Поштучное ценообразование (B2C)**: Цены в каталогах и формах для пользователей всегда должны отображаться за 1 штуку (делением `pricePer1000` на 1000), чтобы розничным клиентам было понятно ценообразование. Допускается формат `₽ / 1 шт.`.

## 5. Качество кода (Developer Excellence)
- **Type Safety**: 100% покрытие TypeScript. Использование `any` запрещено, кроме исключительных случаев интеграции с внешними API.
- **DRY & KISS**: Не дублировать логику. Сложные задачи решать максимально простым способом.
- **Documentation**: Каждая важная функция должна иметь краткое описание. Крупные изменения должны сопровождаться `walkthrough.md`.
