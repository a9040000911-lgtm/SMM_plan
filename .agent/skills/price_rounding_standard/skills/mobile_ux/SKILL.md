# Mobile Architecture & UX Standards

Данный навык определяет правила проектирования и разработки мобильных интерфейсов для Smmplan. Цель — обеспечить пользовательский опыт уровня нативного мобильного приложения (Native-like experience) в веб-браузере.

## 1. Ergonomics & Touch Targets (Правило Большого Пальца)
- **Safe Zones**: Основные интерактивные элементы (кнопки, табы) должны располагаться в нижней трети экрана для удобства управления одной рукой.
- **Minimum Target Size**: Кнопки и ссылки должны иметь область клика не менее 44x44px.
- **Visual Feedback**: Использовать активные состояния (active/tap) для всех тач-элементов (уменьшение масштаба, изменение прозрачности).

## 2. Adaptive Layouts (Fluid & Fixed)
- **Container Strategy**: Использование `max-w-screen-xl` для десктопа и `w-full` для мобильных устройств.
- **Breakpoints**: Приоритет на `mobile (375px+)`, `tablet (768px+)` и `desktop (1280px+)`. Избегать жестких привязок к конкретным моделям устройств.
- **Dynamic Content**: При скрытии десктопных элементов (например, длинных таблиц) на мобильных устройствах, данные должны трансформироваться в карточки или горизонтальные карусели.

## 3. Mobile Navigation (Tab Bar vs Hamburger)
- **Persistent Bottom Bar**: Ключевые разделы приложения (Home, Dashboard, Orders, Profile) должны быть доступны в фиксированной нижней панели навигации (Bottom Tab Bar).
- **Secondary Nav**: Гамбургер-меню используется только для второстепенных настроек и юридической информации.
- **Contextual Actions**: Плавающие кнопки действия (FAB) для критических задач (например, "Создать заказ").

## 4. Performance & Perception
- **Skeleton Screens**: Показ скелетной анимации вместо спиннеров при загрузке контента для снижения уровня тревожности пользователя.
- **Image Optimization**: Использование современных форматов (WebP/AVIF) и адаптивных размеров через `next/image`.
- **Lazy Loading**: Ленивая загрузка тяжелых блоков под скроллом.

## 5. Mobile Component Patterns
- **Bottom Sheets**: Вместо модальных окон в центре экрана использовать выдвижные панели снизу (Bottom Sheets) для настроек, фильтров и описаний услуг.
- **Input Optimization**: Использование правильных `inputmode` (numeric, email, url) для вызова соответствующей клавиатуры на смартфоне.
- **Swipe Actions**: Внедрение жестов смахивания для закрытия уведомлений или переключения табов (где применимо).

## 6. Architectural Integration
- **Client-Side Hydration**: Убедиться, что сложные анимации Framer Motion не блокируют основной поток отрисовки (Main Thread).
- **Z-Index Management**: Четкая иерархия слоев: Content (0) -> Header (10) -> Mobile Nav (50) -> Overlays/Sheets (100).
- **Viewport Locking**: Предотвращение нежелательного зума при фокусе на инпутах в iOS через `meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1"`.
