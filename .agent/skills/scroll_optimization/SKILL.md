# Scroll Optimization & Conversion Maximization (Анти-скролл)

Этот навык предназначен для аудита пользовательских интерфейсов с целью устранения «лишних движений» и минимизации необходимости скроллинга для совершения целевых действий (ввод почты, ссылки, создание тикета, покупка).

## 🛑 Главный принцип: "Конверсия не ждет скролла"
Каждый дополнительный скролл или пиксель, отделяющий пользователя от целевого действия (ввод данных, кнопка "Купить"), математически снижает конверсию (Drop-off rate). Ваша задача — спроектировать UI так, чтобы главный функционал экрана был доступен **Above the Fold** (на первом экране) или с минимально возможным усилием.

## 📋 Чеклист оптимизации скроллов:

### 1. Доступность инпутов (Inputs Accessibility)
- [ ] Поля для ввода ссылки и email видны сразу при загрузке страницы или открытии модального окна. 
- [ ] Клавиатура на мобильных устройствах не перекрывает важные инпуты (использовать `scrollIntoView` при фокусе).
- [ ] Отсутствуют избыточные отступы (`gap`, `padding`, `margin`) между заголовком и формой ввода.

### 2. Вертикальный ритм и плотность (Vertical Rhythm & Density)
- [ ] Максимально уплотненный контент в инструментах для работы (Dashboard, Support, Order Form). Избегайте "маркетинговых" отступов (типа `py-24`) внутри рабочих панелей.
- [ ] Высота шапок (Headers) минимальна в рабочих зонах. Использовать компактные варианты.
- [ ] Большие текстовые блоки скрыты под "Читать далее" или аккордеоны, если они не несут конверсионной ценности.

### 3. Экраны "Создания" и "Настройки" (Modals & Drawers)
- [ ] Формы создания тикета или настройки заказа на мобильных устройствах занимают 100% высоты (Drawer/BottomSheet), но кнопка "Подтвердить" всегда закреплена внизу (Sticky Bottom).
- [ ] Ключевая информация (цена, статус) всегда на виду (Sticky Header или Sticky Footer).
- [ ] Избегать внутренних скроллов (scroll within scroll), когда это возможно. Плавный общий скролл страницы лучше.

### 4. Адаптивность (Responsiveness)
- [ ] Протестированы основные разрешения: 1440px (Десктоп), 768px (Планшет), 375px (Мобильный).
- [ ] На мобильном: элементы размещаются вертикально с `gap-3` или `gap-4` вместо десктопных `gap-8`.
- [ ] На десктопе: инструменты располагаются в 2 колонки (например, форма слева, превью или статус справа), чтобы задействовать горизонтальное пространство вместо вертикального.

## 🚀 Процедура аудита (Workflow):
1. **Сбор данных:** Изучите текущий экран с помощью инструмента browser_subagent. Обратите внимание на высоту элементов.
2. **Анализ отступов:** Проверьте классы Tailwind (`py-`, `my-`, `gap-`, `h-`). Найдите те, которые можно уменьшить без потери эстетики.
3. **Репозиционирование:** Перенесите важные формы выше. Если есть второстепенный текст (`p-text-slate-400`), сделайте его компактнее (`text-xs`).
4. **Закрепление CTA:** Сделайте кнопки действия (`Fixed` или `Sticky`) так, чтобы они всегда были под рукой.
5. **Верификация:** Проверьте, что форма полностью помещается в стандартный экран (высота 800px для десктопа, 700px для мобайла).
