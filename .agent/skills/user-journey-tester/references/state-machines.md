## 📦 Заказ (Order)

### Диаграмма состояний

```
                           ┌─────────────────┐
                           │     PENDING     │
                           │   (Ожидание)    │
                           └────────┬────────┘
                                    │
                    ┌───────────────┼───────────────┐
                    │               │               │
                    ▼               ▼               ▼
            ┌───────────┐   ┌─────────────┐   ┌──────────┐
            │ CANCELLED │   │ IN_PROGRESS │   │  FAILED  │
            │ (Отменён) │   │ (В работе)  │   │ (Ошибка) │
            └───────────┘   └──────┬──────┘   └────┬─────┘
                                 │               │
                    ┌────────────┼────────┐      │
                    │            │        │      │
                    ▼            ▼        ▼      ▼
            ┌───────────┐ ┌──────────┐ ┌──────────┐
            │ COMPLETED │ │ PARTIAL  │ │  REFUND  │
            │ (Выполнен)│ │ (Частич.)│ │ (Возврат)│
            └───────────┘ └────┬─────┘ └──────────┘
                              │
                              ▼
                        ┌──────────┐
                        │  REFUND  │
                        └──────────┘
```

### Матрица переходов

| Из → В | PENDING | IN_PROGRESS | COMPLETED | CANCELLED | FAILED | PARTIAL | REFUND |
|--------|---------|-------------|-----------|-----------|--------|---------|--------|
| **PENDING** | — | ✅ | ❌ | ✅ | ✅ | ❌ | ❌ |
| **IN_PROGRESS** | ❌ | — | ✅ | ✅ | ✅ | ✅ | ❌ |
| **COMPLETED** | ❌ | ❌ | — | ❌ | ❌ | ❌ | ✅ |
| **CANCELLED** | ❌ | ❌ | ❌ | — | ❌ | ❌ | ✅ |
| **FAILED** | ✅ | ❌ | ❌ | ❌ | — | ❌ | ❌ |
| **PARTIAL** | ❌ | ❌ | ✅ | ❌ | ❌ | — | ✅ |
| **REFUND** | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ | — |

### Тест-кейсы для переходов

| TC ID | Начальный | Действие | Конечный | Проверки |
|-------|-----------|----------|----------|----------|
| SM-001 | PENDING | Провайдер принял | IN_PROGRESS | Уведомление USER |
| SM-002 | PENDING | USER отменил | CANCELLED | Возврат средств |
| SM-003 | PENDING | Провайдер не ответил | FAILED | Retry механизм |
| SM-004 | IN_PROGRESS | Выполнено 100% | COMPLETED | Уведомление USER |
| SM-005 | IN_PROGRESS | Выполнено 80% | PARTIAL | Частичный refund |
| SM-006 | IN_PROGRESS | USER отменил | CANCELLED | Частичный возврат |
| SM-007 | COMPLETED | ADMIN refund | REFUND | Полный возврат |
| SM-008 | FAILED | Retry | PENDING | Повторная отправка |

## 💳 Транзакция (Transaction)

### Диаграмма состояний

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   PENDING   │ ──► │  PROCESSING │ ──► │  COMPLETED  │
│  (Ожидание) │     │ (Обработка) │     │ (Завершена) │
└──────┬──────┘     └──────┬──────┘     └─────────────┘
       │                   │
       │                   ▼
       │           ┌─────────────┐
       └──────────►│   FAILED    │
                   │  (Ошибка)   │
                   └─────────────┘
```

### Типы транзакций

| Тип | Описание | Влияние на баланс |
|-----|----------|-------------------|
| DEPOSIT | Пополнение | +сумма |
| WITHDRAW | Списание заказа | -сумма |
| REFUND | Возврат | +сумма |
| BONUS | Бонус | +сумма |
| REFERRAL | Реферальные | +сумма |

### Диаграмма состояний

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│    NEW      │ ──► │    OPEN     │ ──► │   PENDING   │
│   (Новый)   │     │  (Открыт)   │     │(Ожид.ответа)│
└─────────────┘     └──────┬──────┘     └──────┬──────┘
                           │                   │
                           ▼                   ▼
                    ┌─────────────┐     ┌─────────────┐
                    │  ESCALATED  │ ──► │   RESOLVED  │
                    │(Эскалирован)│     │  (Решён)    │

                                               ▼
                                        ┌─────────────┐
                                        │   CLOSED    │
                                        │  (Закрыт)   │
                                        └─────────────┘
```

## 🔄 Drip-Feed заказ

### Диаграмма состояний

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   CREATED   │ ──► │   RUNNING   │ ──► │  COMPLETED  │
│  (Создан)   │     │(Выполняется)│     │ (Завершён)  │
└─────────────┘     └──────┬──────┘     └─────────────┘
       │                   │
       │                   ▼
       │           ┌─────────────┐
       │           │   PAUSED    │
       │           │ (Приостанов)│
       └──────────►└──────┬──────┘
                          │
                          ▼
                   ┌─────────────┐
                   │   STOPPED   │
                   │  (Останов.) │
                   └─────────────┘
```

### Логика дочерних заказов

```
Родительский заказ (Drip-Feed)
├── Дочерний заказ #1 → SCHEDULED → IN_PROGRESS → COMPLETED
├── Дочерний заказ #2 → SCHEDULED → waiting...
├── Дочерний заказ #3 → SCHEDULED → waiting...
├── ...
└── Дочерний заказ #N → SCHEDULED → waiting...

Интервал выполнения: каждый N минут
Статус родителя = агрегация статусов детей
```

## 🧪 Тестирование State Machines

### Общие проверки

Для каждой сущности проверьте:

- [ ] Все переходы по матрице возможны
- [ ] Недопустимые переходы блокируются
- [ ] Атомарность операций (транзакции)
- [ ] Уведомления при смене состояния
- [ ] Логирование всех переходов
- [ ] Recovery после сбоев
- [ ] Параллельные изменения (optimistic locking)

### Скрипт проверки переходов

```python
def test_order_transitions():
    """Тестирует все переходы заказа."""
    transitions = [
        ('PENDING', 'IN_PROGRESS', True),
        ('PENDING', 'COMPLETED', False),  # Недопустимо
        ('IN_PROGRESS', 'COMPLETED', True),
        ('COMPLETED', 'REFUND', True),
    ]
    
    for from_state, to_state, expected in transitions:
        result = can_transition(from_state, to_state)
        assert result == expected, f"Transition {from_state}→{to_state} failed"
```
