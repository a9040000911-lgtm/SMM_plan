# User Journeys: SUPPORT (Техподдержка)

Полный набор сценариев для роли SUPPORT в SMM-панели.

---

## 📋 Обзор модулей SUPPORT

```
SUPPORT доступные модули:
├── 📧 Тикеты (Tickets)
├── 👤 Пользователи (Users)
├── 🛒 Заказы (Orders)
├── 📊 Статистика (Stats)
└── ⚙️ Шаблоны ответов (Templates)
```

---

## Journey S1: Обработка нового тикета

### Happy Path
```
1. SUPPORT входит в систему
   └─→ Видит дашборд с очередью тикетов ✅

2. Видит счётчик новых тикетов: 5
   └─→ Уведомление видно ✅

3. Открывает список тикетов
   └─→ Фильтр "Новые" активен ✅
   └─→ Сортировка по времени ✅

4. Открывает тикет #1001
   └─→ Видит:
       - Тема: "Заказ не выполняется"
       - Автор: user@email.com
       - Дата создания: 2024-01-15 10:30
       - Прикреплённые файлы: screenshot.png
   └─→ Данные корректны ✅

5. Читает описание проблемы
   └─→ Текст читаем ✅

6. Нажимает "Посмотреть заказ"
   └─→ Открывается модальное окно с заказом ✅
   └─→ Видит статус, историю, провайдера ✅

7. Выбирает шаблон ответа: "Стандартный ответ о задержке"
   └─→ Текст подставляется ✅

8. Редактирует ответ
   └─→ Добавляет конкретику ✅

9. Нажимает "Отправить"
   └─→ Ответ отправлен ✅
   └─→ Статус тикета: "Ожидает ответа пользователя" ✅
   └─→ Пользователь получает уведомление ✅
```

### Error Paths
| Сценарий | Ожидается |
|----------|-----------|
| Тикет уже обработан другим агентом | "Тикет был назначен другому агенту" |
| Заказ не найден | "Ошибка загрузки данных заказа" |
| Файл не открывается | "Не удалось открыть вложение" |
| Ошибка отправки | "Ошибка отправки, попробуйте снова" |

## Journey S2: Эскалация тикета

### Happy Path
```
1. SUPPORT открывает тикет #1002
   └─→ Проблема: "Требуется возврат средств" ✅

2. Анализирует ситуацию
   └─→ Видит, что не может решить самостоятельно ✅

3. Нажимает "Эскалировать"
   └─→ Открывается форма эскалации ✅

4. Выбирает уровень: "Администратор"
   └─→ Список админов проекта ✅

5. Выбирает админа: admin@project.com
   └─→ Назначение сохранено ✅

6. Добавляет комментарий: "Требуется refund, превышен мой лимит"
   └─→ Комментарий добавлен ✅

7. Нажимает "Передать"
   └─→ Тикет передан ✅
   └─→ Статус: "Эскалирован" ✅
   └─→ ADMIN получает уведомление ✅
   └─→ SUPPORT больше не видит в своей очереди ✅
```

## Journey S3: Работа с пользователем

### Happy Path
```
1. SUPPORT открывает профиль пользователя
   └─→ Видит:
       - Email, имя, дата регистрации
       - Баланс: 500 RUB
       - Всего заказов: 15
       - Активных заказов: 2
       - История тикетов: 3
   └─→ Данные полные ✅

2. Просматривает историю заказов
   └─→ Список загружается ✅
   └─→ Фильтрация работает ✅

3. Проверяет транзакции
   └─→ История показывает все операции ✅

4. Нажимает "Написать пользователю"
   └─→ Форма сообщения открывается ✅

5. Отправляет сообщение
   └─→ Пользователь получает уведомление ✅
```

### Error Paths
| Сценарий | Ожидается |
|----------|-----------|
| Пользователь заблокирован | Предупреждение "Пользователь заблокирован" |
| Нет прав просмотра профиля | "Недостаточно прав для просмотра" |

## Journey S4: Использование шаблонов ответов

### Happy Path
```
1. SUPPORT открывает "Шаблоны ответов"
   └─→ Список шаблонов загружен ✅

2. Видит категории:
   - Заказы
   - Оплата
   - Технические проблемы
   - Возвраты
   └─→ Категории понятны ✅

3. Выбирает шаблон: "Задержка выполнения заказа"
   └─→ Текст подставляется в ответ ✅

4. Редактирует переменные:
   - {order_id} → 12345
   - {estimated_time} → 24 часа
   └─→ Переменные заменены ✅

5. Создаёт свой шаблон
   └─→ Нажимает "Сохранить как шаблон" ✅
   └─→ Шаблон сохранён ✅
```

## Journey S5: Просмотр статистики

### Happy Path
```
1. SUPPORT открывает "Статистика"
   └─→ Дашборд загружен ✅

2. Видит метрики за период:
   - Обработано тикетов: 150
   - Среднее время ответа: 15 мин
   - Удовлетворённость: 4.5/5
   - Эскалаций: 12
   └─→ Метрики корректны ✅

3. Фильтрует по дате: "Последние 7 дней"
   └─→ Данные обновляются ✅

4. Экспортирует отчёт
   └─→ Файл скачивается ✅
```
