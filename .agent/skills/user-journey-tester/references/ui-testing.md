# UI Testing Methodology

Методология тестирования пользовательского интерфейса с фокусом на кнопки, ожидания vs реальность, и семантику.

---

## 🖱️ Тестирование кнопок

### Чек-лист проверки кнопки

Для каждой кнопки в интерфейсе проверьте:

| # | Проверка | Ожидание | Результат |
|---|----------|----------|-----------|
| 1 | **Видимость** | Кнопка видна на странице | ✅/❌ |
| 2 | **Читаемость текста** | Текст понятен и читаем | ✅/❌ |
| 3 | **Размер** | Кнопка достаточно крупная для клика (min 44px) | ✅/❌ |
| 4 | **Цвет/контраст** | Контраст с фоном ≥ 4.5:1 | ✅/❌ |
| 5 | **Hover эффект** | Визуальная реакция при наведении | ✅/❌ |
| 6 | **Active эффект** | Визуальная реакция при нажатии | ✅/❌ |
| 7 | **Focus состояние** | Видимая рамка при Tab навигации | ✅/❌ |
| 8 | **Disabled состояние** | Корректное отображение неактивной кнопки | ✅/❌ |
| 9 | **Loading состояние** | Индикатор загрузки при действии | ✅/❌ |
| 10 | **Результат клика** | Ожидаемое действие выполняется | ✅/❌ |

### Тест-кейс: Проверка кнопки "Оформить заказ"

```markdown
## TC-BUTTON-001: Кнопка "Оформить заказ"

### Исходное состояние
- Пользователь на странице оформления заказа
- Корзина содержит товары
- Баланс достаточен

### Тестовые сценарии

| Сценарий | Действие | Ожидание | Фактически | Статус |
|----------|----------|----------|------------|--------|
| Hover | Навести курсор | Подсветка, cursor: pointer | | ✅/❌ |
| Click | Нажать кнопку | Запуск процесса заказа | | ✅/❌ |
| Loading | Во время обработки | Spinner, кнопка disabled | | ✅/❌ |
| Success | Успешный заказ | Сообщение, редирект | | ✅/❌ |
| Error | Ошибка обработки | Сообщение об ошибке | | ✅/❌ |
| No balance | Недостаточно средств | Кнопка disabled + tooltip | | ✅/❌ |
| Double click | Быстрый двойной клик | Только один заказ | | ✅/❌ |
| Enter key | Нажатие Enter | Аналогично клику | | ✅/❌ |
| Tab focus | Tab навигация | Фокус на кнопке | | ✅/❌ |
| Mobile tap | Тап на мобильном | Корректная реакция | | ✅/❌ |
```

## ⚖️ Ожидание vs Реальность

### Матрица проверки

Для каждого действия пользователя:

| Действие | Что ожидает пользователь | Что происходит | Соответствие |
|----------|-------------------------|----------------|--------------|
| Клик "Купить" | Товар добавится, уведомление | ? | ✅/❌ |
| Клик "Оплатить" | Переход к оплате | ? | ✅/❌ |
| Клик "Отмена" | Отмена действия, возврат | ? | ✅/❌ |
| Клик "Удалить" | Удаление, подтверждение | ? | ✅/❌ |
| Закрыть модалку | Закрытие без изменений | ? | ✅/❌ |

## 🌐 Семантика и локализация

### Проверка семантики интерфейса

**Принцип:** Названия элементов должны быть понятны русскоязычному пользователю без дополнительных объяснений.

### Чек-лист локализации

| Категория | Проверка | Проблема | Рекомендация |
|-----------|----------|----------|--------------|
| **Кнопки** | Текст понятен? | "Submit" вместо "Отправить" | Использовать русский |
| **Заголовки** | Соответствуют содержимому? | "Панель" вместо "Панель управления" | Уточнить |
| **Ошибки** | Понятна причина? | "Error 500" | "Ошибка сервера, попробуйте позже" |
| **Подсказки** | Помогают пользователю? | "Enter valid email" | "Введите корректный email" |
| **Плейсхолдеры** | Показывают пример? | "Email" | "example@mail.ru" |
| **Успех** | Понятно что случилось? | "OK" | "Заказ успешно создан" |

```markdown
### Участники
- USER (клиент)
- SYSTEM (автоматические процессы)
- ADMIN (администратор)
- SUPPORT (техподдержка)

### Цепочка событий

| # | Роль | Действие | Ожидание | UI элемент | Результат |
|---|------|----------|----------|------------|-----------|
| 1 | USER | Регистрация | Аккаунт создан | Форма регистрации | ✅/❌ |
| 2 | USER | Пополнение баланса | Баланс +1000₽ | Форма оплаты | ✅/❌ |
| 3 | USER | Создание заказа | Заказ в PENDING | Форма заказа | ✅/❌ |
| 4 | SYSTEM | Отправка провайдеру | Статус IN_PROGRESS | Автоматически | ✅/❌ |
| 5 | SYSTEM | Выполнение | Статус COMPLETED | Автоматически | ✅/❌ |
| 6 | USER | Проблема с качеством | Желание сообщить | Кнопка "Поддержка" | ✅/❌ |
| 7 | USER | Создание тикета | Тикет создан | Форма тикета | ✅/❌ |
| 8 | SUPPORT | Получение тикета | Видит в очереди | Дашборд поддержки | ✅/❌ |
| 9 | SUPPORT | Ответ пользователю | Уведомление USER | Форма ответа | ✅/❌ |
| 10 | ADMIN | Просмотр ситуации | Видит историю | Админ-панель | ✅/❌ |
| 11 | ADMIN | Решение (refund) | Возврат средств | Кнопка "Refund" | ✅/❌ |

### Что проверяется на каждом шаге:
- Кнопка/элемент существует и виден
- Действие выполняется корректно
- Результат соответствует ожиданию
- Уведомление/обратная связь получена
- Данные синхронизированы между ролями
```

```python
    return results
```

```python
def check_localization(page):
    """Проверяет локализацию на странице."""
    
    # Русские буквы
    russian_pattern = re.compile(r'[а-яА-ЯёЁ]')

    problem_words = [
        'submit', 'cancel', 'delete', 'confirm', 'login', 
        'register', 'dashboard', 'settings', 'profile',
        'pending', 'processing', 'completed', 'failed'
    ]
    
    issues = []
    
    # Получаем весь текст страницы
    text_content = page.inner_text('body')
    
    # Проверяем на проблемные слова
    for word in problem_words:
        if word.lower() in text_content.lower():
            issues.append(f"Found untranslated word: '{word}'")
    
    return issues
```
