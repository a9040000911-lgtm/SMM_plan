# Visionary UI Engineer Skill (v1.0)

Этот скилл предназначен для создания интерфейсов Smmplan уровня «Wow-эффект 2026». Он игнорирует стандартные решения в пользу «Next-Gen Fintech» эстетики.

## 1. Эстетический манифест: Liquid Glass
*   **Глубина**: Использование `backdrop-blur-3xl` в сочетании с многослойными тенями (`box-shadow: 0 50px 100px -20px rgba(0,0,0,0.25)`).
*   **Грани**: Контейнеры должны иметь 1px границу с 10-20% прозрачностью белого и «верхний блик» (white gradient h-px).
*   **Цвет**: Использование P3 палитры. Вместо обычного Blue-600 — глубокий `indigo-600` или `blue-500` с сиянием.
*   **Типография**: Сочетание мощного `font-black` (Inter/Outfit) для заголовков и `font-serif` (italic) для акцентов.

## 2. UX-Логика: AI Co-pilot
*   **Адаптивный Хаб**: Поле ввода ссылки — это не просто input, это «Аналитический Хаб». При вводе ссылки интерфейс должен «оживать» (Chameleon UI — смена цветовой темы под платформу: Telegram -> Синий, VK -> Голубой).
*   **Прозрачность**: Статусы «AI Анализ...», «Фильтрация провайдеров...» должны быть визуализированы через микро-анимации.
*   **Доверие**: Использование индикаторов «Secure Protocol v4.0» и «AI-Guarantee Verified».

## 3. Технологический стек (Tailwind 4 + Framer Motion)
*   **Container Queries**: Компоненты должны менять вид в зависимости от размера родителя, а не только экрана.
*   **Hyper-Motion**: Анимации входа через `staggerChildren`. Hover-эффекты должны быть «упругими» (`type: "spring", stiffness: 300`).
*   **Zero-UI Elements**: Скрытие второстепенных опций во вложенные меню или динамические подсказки.

## 4. Чек-лист «Strict Client»
- [ ] Есть ли в блоке «воздух» (padding-32+)?
- [ ] Ощущается ли интерфейс «дорогим» (тонкие линии, качественные SVG)?
- [ ] Понимает ли пользователь за 1.5 секунды, что делать дальше?
- [ ] Работает ли правило «The Precision Curator» (иерархия и акценты)?
