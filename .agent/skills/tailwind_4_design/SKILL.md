# Tailwind CSS 4.0

В проекте Smmplan используется **Tailwind CSS v4.0**. Версия 4 кардинально меняет подход к конфигурации.

## 1. Zero Config (Без tailwind.config.js)
- В проекте **НЕ ДОЛЖНО БЫТЬ** файлов `tailwind.config.js` или `tailwind.config.ts`.
- Вся настройка тем, кастомных цветов, шрифтов и брейкпоинтов производится прямо в главном CSS файле (обычно `src/styles/globals.css` или `app/globals.css`) с использованием директивы `@theme`.

### Пример @theme:
```css
@import "tailwindcss";

@theme {
  --font-sans: 'Inter', ui-sans-serif, system-ui, sans-serif;
  --color-primary: #2563eb;
  /* Для динамических цветов */
  --color-brand-500: var(--custom-brand-color, #ff0000);
}
```

## 2. Обновленный синтаксис
- Tailwind 4 использует CSS-переменные для всего. Например, класс `text-primary` будет использовать `--color-primary` из темы.
- Динамические значения (Arbitrary values) работают как и раньше: `w-[200px]`, `bg-[#123456]`.
- Переход градиентов: используйте новые CSS-утилиты (например, возможность смешивания цветов) если это позволяет v4.

## 3. Индивидуальные стили и @apply
- Избегайте создания громоздких классов с помощью `@apply`. 
- Концепция Tailwind 4 — это максимальное использование utility-классов напрямую в JSX (через `cn()` / `clsx` / `twMerge` для условного рендеринга). 

## 4. Производительность (Performance Engine)
Версия 4.0 построена на новом движке (Rust/Lightning CSS), что делает компиляцию мгновенной. Не бойтесь использовать большое количество утилит на странице — это не замедлит сборку. Запрещается дробить стили на отдельные `.css` или `.module.css` файлы без крайней необходимости (оставьте это только для сложной анимации, если Tailwind ломает логику).
