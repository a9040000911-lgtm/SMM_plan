# Zero-Risk Refactoring Protocol (App Router)

Этот навык обязывает ИИ-агента применять сверхконсервативный, безопасный подход при любом изменении структуры папок, файлов или роутинга в Next.js (App Router).

В App Router физическая структура файлов напрямую диктует URL-пути и ломает относительные импорты (`../../`) при любом перемещении. Массовое перемещение файлов — это гарантированная поломка билда и утечка времени. 

## 🛡️ Железные правила безопасного рефакторинга

При поступлении задачи на рефакторинг, реструктуризацию или удаление дубликатов папок, **Агент обязан подчиняться следующему алгоритму:**

### 1. Absolute Git Isolation (Изоляция)
Категорически запрещено проводить крупный рефакторинг в рабочих ветках (`main`, `dev`). Агент обязан:
- Создать изолированную ветку: `git checkout -b refactor/имя-компонента-cleanup`.
- Убедиться, что до начала работ проект компилируется: выполнить `npm run build`.

### 2. Incremental Migration (Инкрементальный перенос)
Запрещено перемещать или удалять более одного функционального домена (модуля) за одну итерацию.
- Если нужно рефакторить 5 папок, процесс разбивается на 5 отдельных атомарных циклов.

### 3. Build Guard (Компилятор-Страж)
Next.js Compiler — самый надежный QA-инженер. После перемещения *каждого* отдельного домена или изменения импортов, Агент обязан:
- Выполнить `npm run build`.
- Если билд упал, Агент не имеет права переходить к рефакторингу следующего домена. Он обязан сначала восстановить "зеленую" сборку (исправить пути, Server Actions).

### 4. Atomic Commits & Reset (Квадратно-Гнездовые сохранения)
Если `npm run build` прошел успешно:
- Немедленно зафиксировать изменения: `git commit -m "refactor(domain): merged X into Y"`.
- Если на следующем шаге билд непоправимо ломается, Агент должен откатиться на последнюю рабочую точку: `git reset --hard HEAD~1` или `git checkout .`.

### 5. Delayed Garbage Collection (Отложенная чистка)
При слиянии модулей (например, перенос старой логики из `/auth` в `/login`), старую пустую папку `/auth` удалять **ТОЛЬКО** в самом конце.
- Перенести файлы.
- Исправить импорты.
- Добиться успешного билда.
- И только после этого выполнить: `rm -rf auth` (или `Remove-Item` в PowerShell).
