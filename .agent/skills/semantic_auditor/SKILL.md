# Semantic Auditor Skill (v1.0)

Скилл для обеспечения лингвистической безупречности и маркетинговой мощности текстов Smmplan.

## 1. Принципы «Магнитных» текстов
*   **Benefit-Driven**: Писать не про "Мы запустили сервис", а про "Вы получили 1000 подписчиков за 2 минуты".
*   **PAS Framework**: 
    1. *Problem* (Проблема — медленный рост).
    2. *Agitation* (Обострение — конкуренты уходят вперед).
    3. *Solution* (Решение — Smmplan).
*   **Jobs to Be Done**: Заголовки должны отвечать на вопрос пользователя: «Какую работу я нанимаю этот сайт выполнить?».

## 2. Технический аудит текста
*   **Harmony**: Длина заголовка не должна превышать 2-3 строки на Desktop и 4 на Mobile.
*   **Logic Check**: Каждое слово должно быть обосновано. Избегать "воды" (лучший, уникальный, инновационный — заменить на конкретику).
*   **Hierarchy**: Заголовок (H1) — эмоция/выгода. Подзаголовок — механика/доверие. CTA — действие.

## 3. Семантическая палитра Smmplan
*   **Тон**: Профессиональный, уверенный, немного технологичный (AI-centric).
*   **Запрещенные слова**: "Накрутка", "Дешево" (заменяем на "Продвижение", "Выгодно/Профессионально").
*   **Ключевые слова**: "Мгновенно", "Гарантия", "AI-фильтрация", "Профессиональный рост".

## 4. Чек-лист аудитора
- [ ] Удалены ли все прилагательные, не несущие смысла?
- [ ] Есть ли в заголовке сильный глагол действия?
- [ ] Соответствует ли текст маркетинговому замыслу (Премиальность)?
