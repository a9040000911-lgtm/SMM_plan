# Skill: Layout Collision Prevention (Правила предотвращения наложений)

При работе с премиальным дизайном, использующим экстремальную типографику (`text-7xl`+) и отрицательные отступы, риск наложения элементов (Layout Collision) возрастает. Этот скилл описывает системный подход к предотвращению таких ситуаций.

## 1. Правило "Дыхания Сетки" (Grid Tension)
**Проблема**: Использование `col-span-6/col-span-6` при огромном тексте слева. Текст `8xl` почти всегда шире 50% контейнера `max-w-6xl`.
**Решение**:
- Для Hero-секций с крупным текстом использовать пропорцию **7/5** или **8/4** (`lg:col-span-7` для текста).
- Всегда закладывать `gap >= 12` (минимум 48px) между колонками.

## 2. Ловушка Bounding Box
**Проблема**: При `leading-[0.85]` и `tracking-tighter` визуальные границы букв (особенно выносные элементы 'ц', 'д', 'й') часто выходят за пределы DOM-контейнера.
**Решение**:
- Не использовать `overflow-hidden` на контейнерах с крупным текстом.
- Использовать `padding-right` вместо `margin-right`, чтобы "забронировать" место для наклона/смещения курсива.

## 3. Опасное позиционирование (Translate-X Trap)
**Проблема**: Использование `translate-x-4` для эстетического сдвига слова без компенсации.
**Решение**:
- Любой положительный `translate-x` должен сопровождаться эквивалентным `pr` (padding-right) или увеличенным зазором сетки.
- Избегать `absolute left-1/2` в узких колонках, если соседний текст имеет непредсказуемую длину.

## 4. Чек-лист проверки (PSSR - Padding, Scale, Split, Rhythm)
1. **P**adding: Есть ли `pr-4` у курсивного заголовка?
2. **S**cale: Не слишком ли велик `8xl` для текущего `max-w`? (На `max-w-6xl` текст `8xl` не должен занимать более 7 колонок).
3. **S**plit: Соответствует ли разделение сетки объему контента? (7/5 против 6/6).
4. **R**hythm: Есть ли `mb-` у абсолютно позиционированных элементов сверху/снизу?
