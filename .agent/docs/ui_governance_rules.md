# Smmplan AI UI Governance & Excellence Rules

Эти правила являются обязательными для исполнения ИИ-агентов при работе над проектом Smmplan.

## 1. Приоритет Визуального Превосходства (Aesthetic-First)
- Интерфейс должен выглядеть как «Премиальный SaaS 2026».
- Использовать: Glassmorphism (`backdrop-blur`), Liquid Gradients, Асимметричные сетки, Динамические тени.
- Избегать: Дефолтных цветов, простых прямоугольников без радиусов скругления, статики.

## 2. Маркетинговая Логика (CRO Engine)
- Каждая кнопка должна иметь «Магнитный эффект» (hover-pulse).
- Первые 3 секунды пользователя на странице должны дать ответы: ЧТО это? ЗАЧЕМ мне это? ПОЧЕМУ сейчас?
- Социальное доказательство (Social Proof) должно быть встроено в каждый значимый блок.

## 3. Техническая Стабильность (Engineering Guard)
- **Линтинг**: Никаких `any`. Все пропсы типизированы.
- **Анимации**: Использовать только `framer-motion`. Никаких `transition-all` в CSS, если это влияет на производительность.
- **Слои**: Запрещено использовать `fixed` без сопровождающей `backdrop-overlay`, если элемент перекрывает интерактивный контент.

## 4. Протокол «Чистый Браузер» (Clean Subagent Rule)
- При проверке интерфейса через `browser_subagent` Агент должен:
  - Удалять системные оверлеи перед скриншотом.
  - Проверять мобильный вид (375px) и десктоп (1440px).
  - Эмулировать медленный интернет для проверки скелетонов.

## 5. Эволюция Навыков
- Агент обязан обновлять свои навыки, если обнаружен новый паттерн дизайна (например, «Bento Grid» или «Mesh Gradients»).
- Ошибки кодирования должны фиксироваться в `lessons_learned.md` для предотвращения регрессий.
