# План улучшения AI Designer: Путь к Zero Artifact UI

Этот план направлен на исключение визуальных багов, наслоения элементов и «кривой» верстки, ориентируясь на стандарты Vercel V0, Bolt.new и Figma AI.

## 1. Внедрение «Визуального Зрения» (Visual Loop)
- **Проблема**: ИИ пишет код, «не видя» результата в реальном времени.
- **Решение**: Каждое изменение в CSS/TSX должно сопровождаться автоматическим скриншотом через `browser_subagent`. Сравнение "Ожидание vs Реальность".
- **Инструмент**: Навык `autonomous_ui_validator`.

## 2. Математика Слоев (Z-Index Hierarchy)
- **Проблема**: Элементы перекрывают друг друга (модалки над кнопками, футеры над контентом).
- **Решение**: Жесткая карта Z-Index (Tokens). ИИ обязан проверять `context` наложения перед `fixed/absolute` позиционированием.
- **Стандарт**:
  - `0`: Content
  - `10-20`: Navigation
  - `50-100`: Hero/Hub elements
  - `1000+`: Global Modals/InstantOrder

## 3. Атомарная Консистентность (Atomic Guard)
- **Проблема**: Использование ad-hoc стилей вместо системных.
- **Решение**: Запрет на использование произвольных значений в Tailwind/CSS. Использование только переменных темы (`--glass-deep`, `--platform-accent`).
- **Стандарт**: Использование `cn()` только с системными утилитами.

## 4. Протокол Самокритики (The "Strict Client" Step)
- **Проблема**: ИИ считает код «выполненным», если он скомпилировался.
- **Решение**: Принудительный этап «Критика». Агент должен запустить браузер, сделать 3 действия и найти 3 изъяна в своем же дизайне перед сдачей.

## 5. Этапы реализации в Smmplan:
1.  **Step 1**: Создание скилла `ui_quality_guard` (Checklist-driven).
2.  **Step 2**: Рефакторинг `HomeContent` и `InstantOrder` с учетом иерархии слоев.
3.  **Step 3**: Внедрение анимационных переходов (Framer Motion Keyframes) для исключения "скачков" контента.

---
*Цель: Достижение уровня дизайна, который ощущается как «отполированный продукт», а не как набор слоев.*
