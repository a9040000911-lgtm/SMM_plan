# Smmplan 🚀

**Smmplan** — это высококонверсионная B2B/B2C SaaS-платформа для управления SMM-услугами, заказами и лояльностью клиентов. Платформа объединяет удобный клиентский интерфейс, мощный Telegram-бот, продвинутую административную панель и интеллектуальную систему предиктивной аналитики.

## 🛠 Технологический стек (Tech Stack)

Проект использует передовые технологии для обеспечения производительности, типизации и современного UI:

- **Фреймворк:** [Next.js 16.0.10](https://nextjs.org/) (App Router, Server Actions)
- **UI Библиотека:** [React 19.0.0](https://react.dev/)
- **Стилизация:** [Tailwind CSS 4.0.0](https://tailwindcss.com/) (с использованием `@theme`)
- **Язык разработки:** [TypeScript 5.7+](https://typescriptlang.org/)
- **Линтинг:** [ESLint 10.0.0](https://eslint.org/) (с использованием Flat Config — `eslint.config.mjs`)
- **База данных:** PostgreSQL + Prisma ORM
- **Telegram Bot:** Telegraf.js (с использованием контекстных сцен и Wizard)

## 🚀 Быстрый старт (Quickstart)

### Предварительные требования (Prerequisites)
Убедитесь, что у вас установлены:
- Node.js (v20+)
- PostgreSQL (или Docker для запуска БД)
- pnpm / npm / yarn (в проекте рекомендуется `npm`)

### Установка

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/your-org/smmplan.git
   cd smmplan
   ```

2. **Установите зависимости:**
   ```bash
   npm install
   ```

3. **Настройте переменные окружения:**
   Скопируйте пример файла конфигурации:
   ```bash
   cp .env.example .env
   ```
   *Далее настройте базу данных, ключи API и Telegram-токена. Полный список смотрите в [Справочнике конфигураций](docs/reference/environment-variables.md).*

4. **Запустите миграции базы данных:**
   ```bash
   npm run prisma:generate
   npx prisma db push
   # Или npx prisma migrate dev
   ```

5. **Локальный запуск (Development):**
   ```bash
   npm run dev
   ```
   *Платформа будет доступна по адресу `http://localhost:3000`.*

## 🤝 Вклад в проект и стандарты работы

Разработка ведется в строгом соответствии с нашими методологиями:
- **Ralph Loop Methodology:** Итеративный фреймворк решения задач "по-умолчанию".
- **Zero-Error Policy:** Обязательная проверка типов (TypeScript) и линтеров перед коммитами.

Полную информацию о том, как правильно отправлять коммиты, создавать ветки и проходить Code Review, читайте в файле **[CONTRIBUTING.md](CONTRIBUTING.md)**.
