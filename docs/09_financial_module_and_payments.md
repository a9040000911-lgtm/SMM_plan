# Модуль: Финансы и Платежи (ЮKassa / Робокасса)

## 💳 Логика пополнения
Мы используем модель **"Депозит"**: пользователь сначала пополняет баланс, а затем списывает его на заказы. Это позволяет мгновенно запускать заказы через API, не дожидаясь подтверждения транзакции от банка в момент покупки.

### Сценарий оплаты:
1. В боте/TMA пользователь нажимает "Пополнить".
2. Выбирает сумму.
3. Система создает транзакцию со статусом `PENDING`.
4. Открывается окно оплаты (ЮKassa).
5. После успешной оплаты ЮKassa отправляет **Webhook** на наш сервер.
6. Сервер обновляет баланс пользователя и меняет статус транзакции на `SUCCESS`.

## 🛡 Безопасность и Налоги
- **Webhook Validation**: Проверка IP-адресов ЮKassa и цифровой подписи запроса.
- **Идемпотентность**: Каждая транзакция имеет уникальный `id`, чтобы исключить двойное зачисление средств при повторном Webhook.
- **Фискализация**: Автоматическая отправка данных для чеков (согласно 54-ФЗ), если используется официальное решение ЮKassa.
