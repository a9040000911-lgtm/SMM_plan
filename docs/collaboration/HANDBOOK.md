# 📔 Памятка Разработчика (Smmplan Handbook)

Этот документ — твой ежедневный помощник. Здесь описано **как именно** делать типичные задачи.

---

## 🏗️ Начало работы

### 1. Как создать задачу (Issue)
1. Зайди в **GitLab -> Issues -> New Issue**.
2. **Title**: Краткое описание (например: "Исправить баг в кнопке 'Купить'").
3. **Description**: Опиши проблему. Добавь скриншот, если есть.
4. **Label**: Выбери `Bug`, `Feature` или `Refactor`.
5. **Assignee**: Назначь на себя.

### 2. Как начать писать код
Когда задача назначена, создай ветку прямо из Issue (кнопка **"Create Merge Request"** в GitLab). 
Это свяжет код и задачу. В терминале на ПК сделай:
```bash
git fetch
git checkout feature/название-вашей-ветки
```

---

## 💻 Процесс разработки

### 3. Как правильно делать коммит
Следуй правилу: Один коммит — одно законченное действие.
```bash
git add .
git commit -m "feat: добавлена валидация ссылок для Telegram"
```
*   `feat`: Новая функция
*   `fix`: Исправление ошибки
*   `docs`: Правка документации

### 4. Перед тем как отправить код (Push)
Обязательно проверь свой код локально:
1. `npm run lint` — есть ли ошибки стиля?
2. `npm run build` — собирается ли проект? (Если он не соберется у тебя, он завалит CI/CD сервер).

---

## 🤝 Взаимодействие и Качество

### 5. Как подтвердить задачу (Merge Request)
Когда код готов и отправлен (`git push`), зайди в GitLab:
1. Проверь "зеленую галочку" (Pipeline passed).
2. Назначь одного из коллег в поле **Reviewers**.
3. Напиши комментарий: "Готово к проверке. Протестировал на таких-то ссылках".

### 6. Чек-лист "Zero Error" (Самопроверка)
Перед тем как позвать коллегу на Review, ответь себе на вопросы:
- [ ] Оставил ли я "грязный" код или `console.log`? (Удали их).
- [ ] Понятны ли названия переменных (не `a`, `b`, а `userBalance`)?
- [ ] Добавил ли я новые ключи в `.env.example`, если они появились?
- [ ] Будет ли это работать на маленьком экране телефона?

---

## 🆘 Если что-то пошло не так

### Конфликты при слиянии (Merge Conflicts)
Если GitLab говорит, что есть конфликты:
1. `git checkout develop` -> `git pull`
2. `git checkout feature/your-branch`
3. `git merge develop`
4. Исправь файлы в редакторе, сохрани и сделай коммит.

### Ошибка сборки на сервере
Если Pipeline стал "красным":
1. Нажми на иконку крестика в GitLab.
2. Прочитай последние строки лога — там будет написана причина ошибки (например, забыт импорт или ошибка типа TypeScript).
