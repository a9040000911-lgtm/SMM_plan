# Руководство по тестированию платежей через ngrok

Чтобы протестировать автоматическое пополнение баланса на локальной машине, платежная система должна иметь возможность прислать уведомление (webhook) на твой адрес.

## 1. Запуск ngrok
Поскольку агенты не могут запускать ngrok напрямую из-за защиты от ботов, тебе нужно сделать это вручную в отдельном терминале:

```bash
ngrok http 3000
```

Скопируй полученный адрес, например: `https://abcd-123.ngrok-free.app`.

## 2. Настройка .env
Открой файл `.env` в корне проекта и обнови следующие переменные, подставив свой адрес ngrok:

```env
NEXT_PUBLIC_URL="https://abcd-123.ngrok-free.app"
WEBAPP_URL="https://abcd-123.ngrok-free.app"
NEXT_PUBLIC_APP_URL="https://abcd-123.ngrok-free.app"
NEXTAUTH_URL="https://abcd-123.ngrok-free.app"
```
> [!IMPORTANT]
> Это необходимо, чтобы после оплаты ЮKassa перенаправляла тебя обратно на ngrok-адрес, а не на localhost, иначе сессия браузера может "потеряться".

## 3. Настройка в кабинете ЮKassa
1. Перейди в раздел **Настройки** -> **Уведомления**.
2. В поле **URL для уведомлений** введи:
   `https://abcd-123.ngrok-free.app/api/webhooks/yookassa`
3. Выбери событие `payment.succeeded`.
4. Сохрани настройки.

## 4. Проведение теста
1. Зайди в свой кабинет V2 по адресу ngrok.
2. Перейди в раздел **Финансы** и нажми **«Пополнить баланс»**.
3. Введи сумму и нажми оплатить.
4. В тестовом режиме ЮKassa используй [тестовые карты](https://yookassa.ru/developers/payment-acceptance/testing-and-go-live/testing#test-cards).
5. После успешной оплаты подожди пару секунд — баланс обновится автоматически.

## 5. Альтернатива: Быстрый симулятор
Если ты не хочешь настраивать кабинет ЮKassa, ты можешь использовать созданный мной скрипт для имитации уведомления:

1. Создай платеж в браузере (дойди до страницы оплаты).
2. Найди ID этого платежа в консоли сервера или в базе данных (таблица `Transaction`, поле `externalId`).
3. Запусти симулятор:
   ```bash
   node scripts/simulate-yookassa.js [ID_ПЛАТЕЖА]
   ```
   *Скрипт использует специальный отладочный заголовок, который я добавил в обработчик, чтобы обойти проверку статуса через API ЮKassa в режиме разработки.*
