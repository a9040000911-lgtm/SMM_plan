# Модуль: Telegram Bot Interface (MVP Interface)

## 🛠 Технологический стек бота
- **Framework**: `telegraf` (Node.js / TypeScript).
- **Session Management**: Redis или Postgres (для хранения состояния заказа).
- **Payments**: Интеграция ЮKassa/Робокасса напрямую через Telegram Payments или через внешнюю ссылку.

## 🤖 Пользовательский сценарий (User Flow)

### 1. Точка входа
Пользователь отправляет ссылку боту.
> **User**: `https://t.me/my_channel`

### 2. Распознавание и Категории
Бот анализирует ссылку и выдает Inline-клавиатуру с категориями.
> **Bot**: 🔍 Ссылка распознана: **Telegram Канал**. Что заказываем?
> [ Подписчики ] [ Просмотры ] [ Реакции ]

### 3. Выбор Тарифа
После выбора категории (например, "Подписчики"), бот показывает доступные тарифы.
> **Bot**: Выберите тариф для **Подписчиков**:
> [ 🥉 Эконом — 49₽/1к ]
> [ 🥈 Стандарт — 89₽/1к ]
> [ 🥇 Премиум — 159₽/1к ]
> [ 👥 Живые (РФ) — 299₽/1к ]
> [ 🌍 Выбрать ГЕО (Другие) ]

### 4. Детализация и Оплата
Пользователь выбирает тариф, вводит количество. Бот проверяет баланс.
> **Bot**: Тариф: **Премиум**.
> Минимум: 100, Максимум: 50 000.
> Введите количество:
> **User**: `1000`
> **Bot**: Итого к оплате: **159₽**. 
> [ Оплатить с баланса ] [ Пополнить баланс ]

## ⚙️ Архитектурные особенности
- **State Machine**: Бот должен четко помнить, на каком этапе находится пользователь (выбор тарифа -> ввод количества).
- **Deep Linking**: Возможность создавать ссылки вида `t.me/smmplan_bot?start=service_123`, которые сразу ведут к заказу конкретной услуги.
- **Уведомления**: Бот присылает сообщения при смене статуса заказа (Запущен / Выполнен).
