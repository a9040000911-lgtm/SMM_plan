# Техническая документация Smmplan 📚

Добро пожаловать в главную директорию инженерной документации проекта **Smmplan**.

Здесь, следуя принципам **Docs-as-Code**, собрана вся структурированная информация об архитектуре проекта, инфраструктуре и правилах взаимодействия с кодом.

Для максимальной удобности восприятия мы используем гибридный подход: адаптированный фреймворк **Diátaxis (Divio)** для разделения документации по "смыслам" и стандарт **arc42 (со схемами Mermaid)** для описания сложной технической архитектуры.

## 🗺 Навигация по документации

### 1. 🎓 Обучающие руководства (Tutorials)
*Для тех, кто впервые знакомится с проектом.*
Раздел ориентирован на обучение и достижение первого успешного результата.
- 📍 [docs/tutorials/local-setup.md](tutorials/local-setup.md) — Пошаговое локальное развертывание Smmplan с нуля (БД, Next.js, Telegram Bot).

### 2. 🛠 Практические инструкции (How-to Guides)
*Для тех, кто решает конкретную профессиональную задачу.*
Готовые рецепты "Как сделать Х". Подразумевает, что читатель уже знаком с основами платформы.
- 📍 [docs/how-to/configure-providers.md](how-to/configure-providers.md) — Добавление и настройка SMM-провайдеров.
- 📍 [docs/how-to/manage-finances.md](how-to/manage-finances.md) — Настройка Unit-экономики, мультивалютности (RUB/USD) и комиссий эквайринга.

### 3. 📖 Справочная информация (Reference)
*Сухие факты, API-спецификации и списки команд.*
Ориентировано исключительно на быстрое получение факта (как справочник).
- 📍 [docs/reference/environment-variables.md](reference/environment-variables.md) — Полный список всех `.env` переменных проекта и их назначение.
- 📍 [docs/reference/api-specification.md](reference/api-specification.md) — Описание маршрутов API и их Zod-схем.

### 4. 🏛 Архитектура проекта (Architecture)
*Отвечает на вопросы "Почему это сделано именно так?" и "Из чего состоит?".*
Создано на базе облегченной версии фреймворка **arc42** с использованием диаграмм **Mermaid.js**.
- 📍 [docs/architecture/01-context.md](architecture/01-context.md) — Бизнес-контекст и взаимодействие с внешним миром (System Context по C4).
- 📍 [docs/architecture/02-building-blocks.md](architecture/02-building-blocks.md) — Обзор архитектурных слоев и основных сервисов (OrderProcessor, ProviderService).
- 📍 [docs/architecture/03-security-and-ux.md](architecture/03-security-and-ux.md) — Описание философии безопасности приложения и подходов Neuro-UX дизайна.
- 📍 Папка `adr/` — Журнал архитектурных решений (Architecture Decision Records).
