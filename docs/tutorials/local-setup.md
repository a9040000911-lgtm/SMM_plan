# 🎓 Обучающее руководство: Локальное развертывание Smmplan с нуля

Это руководство проведет вас по шагам, необходимым для запуска проекта **Smmplan** на вашем локальном компьютере. Если вы запускаете проект впервые, просто следуйте шаг за шагом — и вы добьетесь успеха.

## Предварительные требования (Prerequisites)

Прежде чем начать, убедитесь, что на вашей машине установлено:

- `Node.js` (рекомендуемая версия v20 LTS).
- `git` (для клонирования проекта).
- `PostgreSQL` (можно установить локально или запускать через Docker/Supabase).
- Удобный редактор кода (например, VS Code с плагинами для ESLint, Tailwind CSS и TypeScript).

## Шаг 1: Клонирование и зависимости

Откройте терминал и склонируйте репозиторий:

```bash
git clone https://github.com/your-org/smmplan.git
cd smmplan
```

Текущая база кода оптимизирована для использования `npm`. Установите все библиотеки и пакеты:

```bash
npm install
```

> [!NOTE]
> Заметили ошибки ERESOLVE при установке? Обычно `npm install --legacy-peer-deps` решает проблему конфликтов версий Next.js и React 19, однако в нашем проекте все версии уже согласованы.

## Шаг 2: Настройка базы данных и переменных окружения

Smmplan хранит все конфиденциальные данные (ключи, доступы) в файле `.env`, который не попадает в систему контроля версий Git. 

Создайте файл окружения на основе примера:

```bash
cp .env.example .env
```

Затем откройте `.env` и настройте подключение к базе данных. Основной параметр — это `DATABASE_URL`.

**Пример (если у вас локальный PostgreSQL):**
```env
DATABASE_URL="postgresql://postgres:password@localhost:5432/smmplan_dev?schema=public"
# Не забудьте указать тестовый токен вашего Telegram-бота:
TELEGRAM_BOT_TOKEN="123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11"
```

> [!TIP]
> Узнать все значения и их смысл можно в нашем [Справочнике конфигураций](../reference/environment-variables.md).

## Шаг 3: Миграция Prisma
Smmplan использует Prisma ORM (Object-Relational Mapping). Нам нужно синхронизировать схему БД, которая лежит в `prisma/schema.prisma`, с вашей локальной БД.

Для разработки запустите:

```bash
# Генерирует Prisma Client:
npm run prisma:generate

# Применяет схему к базе данных (создаст таблицы):
npx prisma db push
```

## Шаг 4: Запуск проекта в режиме разработки

Всё готово для старта! Запускайте приложение:

```bash
npm run dev
```

В консоли вы увидите: `ready - started server on [::]:3000, url: http://localhost:3000`.

- **Web UI:** Откройте браузер и перейдите по адресу `http://localhost:3000`. Вы должны увидеть красивый интерфейс (используйте темную тему!).
- **Telegram Bot:** Если вы указали верный `TELEGRAM_BOT_TOKEN`, бот тоже будет готов отвечать на ваши команды (`/start`).

> [!IMPORTANT]
> **Авторизация.** По умолчанию система работает на Session Auth (или NextAuth). Если вы тестируете админ-панель (`/admin`), убедитесь, что ваш тестовый JWT-токен или сессия имеет роль `ADMIN`. Вы можете изменить свою роль прямо в базе данных с помощью Prisma Studio (`npx prisma studio`).

Поздравляем! 🎉 Вы успешно развернули **Smmplan**. Теперь изучите структуру папок или переходите к [инструкции по настройке SMM-провайдеров](../how-to/configure-providers.md).
