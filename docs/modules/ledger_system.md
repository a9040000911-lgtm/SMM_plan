# Module: Ledger System & Reconciliation

## 📍 Назначение
Обеспечение 100% финансовой прозрачности и защита от несанкционированного изменения балансов.

## 🧱 Структура данных
- **LedgerEntry**: Неизменяемая запись каждой финансовой операции. Хранит состояние баланса "ДО" и "ПОСЛЕ".
- **Transaction**: Журнал внешних взаимодействий (платежные системы, API провайдеров).

## 📋 Функционал
- **Double-Entry Verification**: Каждое изменение баланса в таблице `User` сопровождается записью в `LedgerEntry`.
- **Automatic Audit**: Фоновый процесс сверяет `User.balance` с суммой всех `LedgerEntry`. При расхождении более 0.1 RUB генерируется алерт.
- **Multi-currency Support**: Автоматическая конвертация исторических записей при сверке, если пользователь сменил валюту профиля.
